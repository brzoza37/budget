# Testing Quality Design
**Date:** 2026-05-03

## Goal

Add PHPUnit test coverage (unit + integration), introduce PHPStan level-9 static analysis, and enforce both on every pull request via GitHub Actions CI. Master is branch-protected; merging is blocked when CI fails.

---

## 1. New Unit Tests (pure, mocked — no DB)

### `tests/Unit/Service/RecurringEventGeneratorServiceTest.php`

Tests the date computation logic in `RecurringEventGeneratorService` using PHPUnit mocks for `PlannedItemRepository` and `EntityManagerInterface`.

| Test case | What it verifies |
|---|---|
| `nextDateDays` | `+N days` advances date correctly |
| `nextDateWeeks` | `+N weeks` advances date correctly |
| `nextDateMonths` | `+N months` advances date correctly |
| `nextDateYears` | `+N years` advances date correctly |
| `pinToDayEndOfMonth` | Jan 31 + 1 month → Feb 28 (not Feb 31) |
| `pinToDayLeapYear` | Jan 31 + 1 month on leap year → Feb 29 |
| `generateIdempotent` | When `existsForEventOnDate` returns true, no item created and `flush` not called |
| `generateReturnsCount` | Returns correct number of new items created |
| `generateFlushesOnce` | `em->flush()` called exactly once when items are created |

### `tests/Unit/State/BudgetStateProviderTest.php`

Tests `BudgetStateProvider` using mocks for `ProviderInterface`, `TransactionRepository`, and `Security`.

| Test case | What it verifies |
|---|---|
| `collectionCallsAttachSpentOnEach` | `getSpentForBudget` called once per budget in collection |
| `itemCallsAttachSpent` | `getSpentForBudget` called for single budget item |
| `nullCategorySkipsSpent` | Budget with null category never calls `getSpentForBudget` |

## 3. Static Analysis

**Tool:** PHPStan

**Packages (dev):**
- `phpstan/phpstan`
- `phpstan/extension-installer`
- `phpstan-symfony/phpstan-symfony`

**Config `backend/phpstan.neon`:**
```neon
includes:
    - vendor/phpstan/extension-installer/src/Config/config.neon

parameters:
    level: 9
    paths:
        - src
        - tests
    symfony:
        containerXmlPath: var/cache/dev/App_KernelDevDebugContainer.xml
```

Level 9 (maximum). Any existing violations are fixed as part of this work — no baseline file.

The Symfony extension uses the compiled DI container XML to resolve `$this->getUser()` return types, `getContainer()->get(...)` types, and similar Symfony-specific patterns. The container XML is generated by `bin/console cache:warmup --env=dev` (no database required).

## 4. GitHub Actions CI Workflow

**File:** `.github/workflows/ci.yml`  
**Trigger:** `pull_request` targeting `master`  
**Structure:** Two parallel jobs.

### Job: `static-analysis`

```
ubuntu-latest
PHP 8.3 (intl, pdo_pgsql), no DB service
composer install (cached on composer.lock)
bin/console cache:warmup --env=dev
vendor/bin/phpstan analyse --no-progress
```

### Job: `tests`

```
ubuntu-latest
PHP 8.3 (intl, pdo_pgsql)
PostgreSQL 16 service (user/password/budget_test, health-checked)
composer install (cached on composer.lock)
bin/console lexik:jwt:generate-keypair  (JWT_PASSPHRASE from GitHub secret)
vendor/bin/phpunit --configuration phpunit.ci.xml
```

JWT_PASSPHRASE must be stored as a GitHub Actions repository secret. The keypair is generated fresh each run and never committed.

**`phpunit.ci.xml` (new file, committed):** `phpunit.dist.xml` uses `db:5432` (Docker hostname) with `force="true"`, which overrides any OS env var — so a separate config file is the only way to change the hostname for CI. `phpunit.ci.xml` is identical to `phpunit.dist.xml` but with `DATABASE_URL` pointing to `localhost:5432`.

**`APP_SECRET` for `static-analysis` job:** `bin/console cache:warmup` requires `APP_SECRET` to be set. A dummy value (`APP_SECRET=ci` or similar) is set as a step env var in the workflow — it is never used beyond DI container compilation.

### Composer scripts added

`composer.json` gets two convenience scripts:
```json
"phpstan": "vendor/bin/phpstan analyse",
"test":    "vendor/bin/phpunit"
```
